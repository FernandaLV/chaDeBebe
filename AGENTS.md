# AGENTS.md

## Project

`chaDeBebe` - baby shower app. Currently scaffolding only (no application code yet).

## Workflow

- Uses **OpenSpec** for spec-driven development (`openspec/`).
- OpenSpec skills are installed under `.opencode/skills/`. Use the slash commands (`/opsx-propose`, `/opsx-apply`, etc.) for the change workflow rather than making changes directly.

## Conventions

- OpenSpec config: `openspec/config.yaml`. Keep `context` fields up to date as the tech stack solidifies.
- Specs live in `openspec/specs/`, changes in `openspec/changes/`.
